ecommerce-platform.tsx

E-Commerce Platform Rebuild

A full storefront and checkout rebuild for a mid-size retailer.

Next.js Node.js Stripe

how it was built

Design: ran two discovery workshops with the retailer, then wireframed and prototyped the new checkout flow in Figma, user-testing it with five shoppers before writing a line of code. Build: implemented the storefront in Next.js with a Node/PostgreSQL backend, integrating Stripe for payments and adding automated tests around checkout edge cases. Delivery: shipped behind a feature flag, rolled out gradually, and monitored conversion metrics post-launch — cart abandonment dropped 18%.

analytics-dashboard.tsx

Real-Time Analytics Dashboard

A live operations dashboard streaming metrics over WebSockets.

React D3.js WebSockets

how it was built

Design: shadowed the ops team for a day to learn which numbers they actually glance at under pressure, then sketched a low-clutter dashboard layout reviewed with three on-call engineers. Build: built custom D3 charts inside a React shell, with a WebSocket layer streaming live metrics and a fallback to polling when sockets drop. Delivery: deployed to a staging environment for a two-week soak test before going live to the full ops team.

api-gateway.go

Banking API Gateway

A secure microservice gateway fronting a banking app's internal APIs.

Go Docker OAuth2

how it was built

Design: mapped every internal service the gateway would front, then drafted an authentication and rate-limiting spec reviewed in a security audit before implementation began. Build: wrote the gateway in Go for throughput, containerised it with Docker, and added OAuth2-based service auth with per-client rate limits. Delivery: rolled out region by region with real-time alerting, catching one mis-configured rate limit before it affected customers.

devkit-cli.rs

Open-Source CLI Tool

A developer CLI for scaffolding and linting internal services.

Rust Open Source DX

how it was built

Design: collected pain points from internal engineers via a short survey, then designed the command structure around the three tasks they repeated most often. Build: built the CLI in Rust for fast startup time, with unit tests for every command and generated docs published alongside each release. Delivery: open-sourced it, set up GitHub Actions for automated releases, and it's now used by several teams beyond the original one.
